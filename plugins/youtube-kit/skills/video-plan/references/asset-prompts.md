# 이미지·영상 에셋 생성 프롬프트 템플릿 (asset-prompts)

샷 리스트에 넣을 **화면 재료**를 직접 만들고 싶은 사람용. 참가자가 ChatGPT·Gemini·Higgsfield 같은 생성 서비스를 쓸 때만 만든다. **안 쓰면 "클로드 기본 이미지로 간다"** 고 안내하고 이 단계는 건너뛴다.

> 형식 원칙: **샷별로 ① 영어 프롬프트(서비스에 그대로 붙여넣기) + ② 한국어 설명(무엇을·왜)** 한 쌍. 영어 프롬프트는 참가자가 손 안 대고 복붙만 하면 되게 완성형으로.

## 화면비를 먼저 정한다 (한 줄)
- **가로 일반 영상:** `16:9` (예: 1920×1080) — 데스크톱·TV에서 크게 보인다.
- **세로 쇼츠:** `9:16` (예: 1080×1920) — 폰에서 꽉 찬다.
- **인터뷰 6번(길이·형태)에서 고른 가로/세로를 그대로 따른다.** 안 정했으면 레퍼런스 채널을 보고 맞추고(기본 **16:9**), 기획서에 "쇼츠로 올릴 거면 프롬프트 끝 `16:9`만 `vertical 9:16`으로 바꿔 다시 생성" 한 줄을 남긴다.
- 프롬프트 끝에 화면비를 **반드시 명시**한다(`16:9` 또는 문장으로 "vertical 9:16").

## 이미지 프롬프트 템플릿 (ChatGPT·Gemini 등)
```
[subject], [setting/background], [mood/lighting], [style: photo/illustration/3d],
clean composition, no text, high detail, 16:9 aspect ratio
```
- 한국어 설명 예: "샷 2번 배경 — 따뜻한 햇살 든 카페 카운터. 글자 없이(자막은 따로 얹음)."
- ⚠️ **이미지 안에 글자를 넣지 말 것**(`no text`) — 자막은 HyperFrames가 따로 얹는다. 글자가 박힌 이미지는 자막과 겹쳐 지저분해진다.

## 영상(움직이는 컷) 프롬프트 템플릿 (Higgsfield 등)
```
[subject doing action], [camera move: slow push-in / pan], [mood], 
cinematic, smooth motion, 5 seconds, 16:9
```
- 한국어 설명 예: "샷 4번 — 손이 메뉴를 가리키는 5초 클립, 천천히 줌인."
- 짧은 컷(3~6초) 여러 개가 긴 한 컷보다 1분 영상에 잘 맞는다.

## 샷↔프롬프트 묶는 법
샷 리스트의 각 행에 프롬프트를 1:1로 붙인다.
| 샷 # | 화면 | 에셋 프롬프트(영어) | 한국어 설명 |
|---|---|---|---|
| 1 | 훅 자막 + 배경 | `cozy cafe counter, morning light, photo, no text, 16:9` | 첫 3초 배경 |
| 2 | … | … | … |

## 사실·안전 규칙
- 실제 인물·상표·로고를 흉내 내는 프롬프트는 피한다(저작권·초상권). 일반 장면·사물로.
- 참가자가 가진 **실제 사진/영상이 있으면 그걸 우선** 쓴다 — 생성은 빈 칸을 메우는 용도.
- 생성 서비스가 없거나 막히면 **클로드 기본 이미지/단색 배경+자막**으로도 충분히 한 편이 된다. 멈추지 않는다.

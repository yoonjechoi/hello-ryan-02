# 내 유튜브 첫 영상 런치 키트 자동 제작 (build-youtube-launch)

완전 초보의 **"내 유튜브 첫 1분 영상"** 을, 따뜻한 인터뷰로 끌어내 **바로 찍을 기획서**를 만들고, 그 결로 **유튜브 채널 얼굴(이름·핸들·설명·이미지)** 까지 한 벌로 맞춘다. ep02 [[build-brand-kit]]의 구조를 그대로 잇되, 결과물은 **오늘 올릴 영상 한 편 + 채널 세팅**이다.

## 두 가지 약속
1. **단계마다 멈춰 확인받고 중간 산출물을 보여준다.** (게이트)
2. **교육 목적** — 단계가 끝나면 "직접 하려면 이렇게 쳐보세요" 프롬프트를 보여준다.

## 우리가 넘는 악당
**"유튜브 첫 시작의 두려움"** — 편집이 무섭고, 그냥 찍자니 부끄럽고, 그래서 첫 발이 안 떨어진다. **첫 시작이 가장 어렵다.** 이 키트는 그 첫 발을 대신 떼 준다. 오늘은 "강제로" 다 같이 시작한다.

## 큰 메시지 (마무리에서) ⭐
"나도 개발자라 내가 올리는 정보가 다 아는 내용 같아 도전이 어려웠다. 동생은 '이런 걸 왜 올리냐' 했지만, 반응은 너무 좋았다. **나에겐 안 중요한 정보가 남에겐 귀중할 수 있다.** 여러분 경험 중 무언가는 분명 누군가에게 귀하다. 한 명이라도 도움이 되면 올릴 값어치가 있다 — 그 한 명이 삼성·LG·현대로 갈지 모른다. **나를 필요로 하는 한 명을 위해, 오늘 1분짜리 영상을 만들자.**" (감동 마무리 — 라이언 스토리 결.)

## Phase 0: 컨텍스트 확인 (초기 / 후속 / 부분 재실행)
슬러그를 정한다(이름/주제 → 영문, 예: `nyc-realty`). 작업 폴더 **`_workspace/youtube-kit/<slug>/`**.
- 폴더 없음 → 초기 실행(1단계부터).
- 폴더 있음 + "대본만 다시/채널만 다시" → 해당 단계만(앞 산출물 재사용).
- 폴더 있음 + 새 인터뷰 → 기존을 `<slug>_prev/`로 옮기고 새로.

부분 재실행: |요청|1 인터뷰|2 영상 기획|3 채널 세팅| / "인터뷰 보완"→1 / "대본·샷 다시"→2 / "채널만 다시"→3.

## 실행 모델
- **1단계는 메인 대화(실시간)** — 참가자와 주고받음. `video-plan` 스킬(1부)대로 리더(=`video-interviewer`)가 직접.
- **2단계는 서브에이전트**(`Agent`, **model: "opus"**) — 대본·샷·에셋 프롬프트는 무거우니 **반드시 서브로**(메인 대화가 안 멈추게).
- **3단계는 혼합:** 방향성·레퍼런스 **선택은 메인 대화(실시간)**, 채널값·이미지 프롬프트 **생성만 서브에이전트**.
  - ⚠️ **선택을 서브에이전트에 넘기지 말 것.** 서브는 참가자와 실시간 대화를 못 해 "방향성 골라라"를 못 묻고 **자동 선택돼 버린다(brand-kit v0.0.3 실제 버그).** 선택은 메인에서 끝내고 고른 결과만 넘긴다.
- 데이터는 **파일로 전달**: `_workspace/youtube-kit/<slug>/`.

## Phase 1 — 영상 인터뷰 (메인 대화)
**스킬:** `video-plan`(1부) · **담당:** 리더 = `video-interviewer`
1. 따뜻하게 한 턴씩 → 주제·청중·원하는 반응·레퍼런스·톤·길이·녹음/에셋 형편을 끌어낸다.
2. 대화 원본을 `00_interview-transcript.md`로 저장(말투 보존·윤문 X) + 맨 아래 '얻은 일곱 결' 매핑.
3. 보여주고 "고칠 데 있으세요?" (게이트 ①).
4. 교육: "'내 유튜브 영상 기획해줘'라고 치면 돼요."

## Phase 2 — 영상 기획서 (서브에이전트)
**스킬:** `video-plan`(2부) · **에이전트:** `video-planner` (**model: "opus"**)
1. `Agent(subagent_type:"video-planner", model:"opus")`로: `00_interview-transcript.md`를 읽어 **1분(또는 30초) 영상 기획서** `01_video-plan.md`를 만든다 — ①주제·길이 고정 ②3초 훅 ③레퍼런스 톤(수치 날조 금지) ④구어체 대본 ⑤샷 리스트 ⑥(선택)에셋 프롬프트 ⑦녹음 안내(+복식호흡 카드).
2. 대본·샷·녹음 안내를 보여준다 (게이트 ②: "이대로 찍어볼 만해요? 더 짧게/훅 더 세게?").
3. 교육: "'대본만 다시 써줘'처럼 한 부분만도 고쳐요."

## Phase 2.5 — 경쟁사 분석 → 모션 프리프로덕션 (빌드 설계, 서브에이전트) ⭐
**스킬:** `video-plan`(3부) · 무거운 조사·설계라 **서브에이전트(opus)**.
1. **경쟁사 분석** → `competitor-analysis.md`: 같은 니치 **실제 상위 영상**의 훅·페이싱·자막·**음악/사운드**·CTA 패턴. ⚠️ 조회수·구독자는 확인된 값만(가능하면 `yt-dlp`), 모르면 안 적음(날조 금지).
2. **모션 프리프로덕션** → `motion-preproduction.md`: 분석 + `npx hyperframes catalog`(쓸 블록) + 브랜드 무드 → **샷별 "무엇이 어떻게 움직이나"**(켄번스 줌·소프트 디졸브·자막 스타일·모션그래픽 블록·BGM/SFX). 레시피: `video-plan/references/motion-preproduction.md`.
3. 보여주고 게이트("이 움직임 결 맞아요?"). → **라이브 HyperFrames 빌드는 샷 리스트가 아니라 이 문서를 따른다.**

## Phase 3 — 채널 세팅 (선택 메인 + 생성 서브)
**스킬:** `channel-setup` · **에이전트:** `channel-setup-maker` (**model: "opus"**)

**3a. 선택 (메인 대화 — ⚠️ 서브에이전트로 넘기지 말 것):**
1. `01_video-plan.md`를 바탕으로 **채널 방향성 후보 2~3개**(+있으면 레퍼런스 결)를 이유 한 줄과 함께 제시 → **참가자가 하나 고른다.** 반드시 답을 기다린다.

**3b. 생성 (서브에이전트):**
2. **고른 방향성을 입력으로** `Agent(subagent_type:"channel-setup-maker", model:"opus")`: 채널명·핸들·설명·키워드/태그·카테고리·방향성 한 줄 + **이미지 프롬프트(프로필 800×800 · 배너 2560×1440/안전영역 1546×423 · 썸네일 1280×720)** → `02_channel-setup.md`.
3. 값·프롬프트와 **어디에 붙여 넣는지**(유튜브 스튜디오 맞춤설정)를 보여준다 (게이트 ③: "이 채널 얼굴 맞아요?").
4. 교육: "'채널명 다시 지어줘'처럼 한 칸만도 바꿔요."

## 마무리
1. 손에 남은 것 정리: **영상 기획서(대본·샷·녹음 안내) + 채널 세팅값**. → 폰/​TTS로 **대본을 녹음해 오면** 라이브 당일 끝.
2. 예고: **"라이브에서 이 대본·샷·녹음을 HyperFrames에 넣으면 자막·자료 화면까지 묶여 한 편이 됩니다."** (편집은 AI가 — 악당을 이기는 법.) 사람 녹음은 **1.2배속으로 살짝 조여** 또렷하게 한다. 빌드는 **`motion-preproduction.md`대로** — 켄번스로 스틸을 움직이고, 카탈로그 모션그래픽을 얹고, BGM·SFX로 소리를 채운다.
3. **큰 메시지(위 라이언 스토리)** 로 닫는다 — "나를 필요로 하는 한 명을 위해 오늘 첫 발을."

```
[메인]  Phase3a 방향성 선택
[channel-setup-maker·서브] 01_video-plan + 선택 → 02_channel-setup.md (채널값·이미지 프롬프트)
```
중간물 `_workspace/youtube-kit/<slug>/` 보존.

## 에러 핸들링
- 서브에이전트 실패 시 1회 재시도, 그래도 실패면 그 산출물 없이 멈추고 보고(임의 추정 금지).
- 앞 단계 부실(예: `00_interview-transcript.md` 빈약) → 다음으로 안 넘기고 보완 권유.
- 레퍼런스 채널 수치는 **모르면 비운다**(조회수·구독자 날조 금지, 추정은 '추정' 표기).
- HyperFrames·유튜브 규격은 **사전 준비 가이드의 검증된 값**만 — 새 명령·새 수치 지어내기 금지.
- 사실 보존: `00_interview-transcript.md`에 없는 경력·수치·고유명사 금지(가상은 "데모용 가상" 표기).

## 모델·윤문 규칙
- 모든 `Agent` 호출 **model: "opus"**. 참가자에게 보이는 한국어 산문(대본·채널 설명·안내)은 `humanize-korean`으로 다듬는다 — 단 **명령어·URL·핸들·규격 수치·고유명사는 보존.** 대본의 의도된 짧은 구어 리듬은 살린다.

## 테스트 시나리오
- **정상:** "민수 님 유튜브 영상 기획해줘" → 슬러그 `nyc-realty` → Phase1 인터뷰(주제=뉴욕 부동산) → 게이트① OK → Phase2 `video-planner`가 3초 훅+1분 대본+샷+폰 녹음 안내 → 게이트② OK → Phase3a 방향성 택1 → 3b `channel-setup-maker`가 채널명·핸들·설명·이미지 프롬프트 → 붙여넣는 법 안내 → 라이언 스토리 마무리. 단계마다 교육 프롬프트.
- **에러:** Phase2에서 레퍼런스 채널 정보 접근 불가 → 일반 고리텐션 흐름 + '추정' 표기로 진행.
- **부분 재실행:** "대본만 30초로 다시" → Phase0이 폴더 감지 → Phase2만 재실행(채널 세팅 보존).
